# Cloud_Mirror

Сервис синхронизации файлов между локальной папкой на компьютере и облачным хранилищем.

## Возможности

- Автоматическая синхронизация локальной папки с облачным хранилищем
- Отслеживание новых, изменённых и удалённых файлов
- Настраиваемый интервал синхронизации
- Логирование всех операций в файл

## Структура проекта

```
Cloud_Mirror/
│
├── core/
│   ├── __init__.py

│
├── utils/
│   ├── __init__.py

├── config.ini              # Конфигурация (не хранится в репозитории)

├── requirements.txt        # Зависимости проекта
├── .gitignore
└── README.md
```

## Установка

1. Клонируйте репозиторий:

```bash
git clone https://github.com/ваш_username/Cloud_Mirror.git
cd Cloud_Mirror
```

2. Создайте и активируйте виртуальное окружение:

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

3. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Настройка

Скопируйте пример конфигурации и заполните своими данными:

```bash
cp config.ini.example config.ini
```

Откройте `config.ini` и укажите параметры:

```ini
[Settings]
local_folder = C:\Users\User\Documents\MyFolder
cloud_folder = MyBackup
token = ВАШ_ТОКЕН_ЗДЕСЬ
sync_interval = 60
log_file = C:\CloudMirror\Logs\sync.log
```

| Параметр        | Описание                                          |
|-----------------|---------------------------------------------------|
| `local_folder`  | Путь к локальной папке для синхронизации          |
| `cloud_folder`  | Название папки в облачном хранилище               |
| `token`         | Токен доступа к Яндекс.Диску                      |
| `sync_interval` | Интервал синхронизации в секундах                 |
| `log_file`      | Полный путь к файлу лога                          |

> ⚠️ **Важно:** никогда не добавляйте `config.ini` с реальным токеном в репозиторий. Файл уже добавлен в `.gitignore`.

## Запуск

```bash
python main.py
```

При запуске программа:
1. Читает параметры из `config.ini`
2. Проверяет токен и доступность локальной папки
3. Проверяет существование папки в облачном хранилище
4. Выполняет первичную синхронизацию
5. Повторяет синхронизацию с заданным интервалом

## Логирование

Все операции записываются в файл, путь к которому указан в `log_file`. Пример записей:

```

CloudMirror 2026-01-15 10:00:01,132 INFO Файл document.pdf успешно записан.
CloudMirror 2026-01-15 10:01:05,489 INFO Файл notes.txt успешно перезаписан.
CloudMirror 2026-01-15 10:02:10,183 INFO Файл old_file.docx успешно удалён.
CloudMirror 2026-01-15 10:03:00,165 ERROR Файл image.png не перезаписан. Ошибка соединения.
```
